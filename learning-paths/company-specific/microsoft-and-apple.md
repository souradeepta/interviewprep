# Microsoft & Apple Interview Prep

**Level:** L3-L5
**Time to read:** ~20 min

Quick guides for Microsoft and Apple technical interviews.

---

## 🟦 Microsoft (SDE1-SDE3)

### Interview Format
```
Phone Screen: 1 coding problem (30-45 min, medium)
Onsite: 2-3 coding rounds (60 min each, medium-hard)
System Design: (Optional, for senior roles)
Behavioral: Integrated into coding rounds
```

### Microsoft-Specific

**Emphasis:**
- Strong fundamentals
- Optimization (time & space)
- Practical problem-solving
- Less on system design for SDE1-2

**Favorite Topics:**
1. **String manipulation** (30%)
   - Substring problems
   - Pattern matching
   - Transformations

2. **Trees/Graphs** (25%)
   - Binary trees
   - Graph traversals
   - Path problems

3. **Arrays** (20%)
   - Sorting
   - Searching
   - Two-pointer

4. **Hash maps** (15%)
   - Optimization tool
   - Frequency problems

5. **DP** (10%)
   - Optimization sequences
   - Subset problems

### Preparation (4 weeks)

```
Week 1: String + Array fundamentals (15 problems)
Week 2: Trees + Graphs (20 problems)
Week 3: Hash maps + DP (15 problems)
Week 4: Mock interviews (3), weak areas
```

### Tips

- **Communicate clearly** (Microsoft values clear thinking)
- **Optimize step-by-step** (naive → optimized)
- **Consider edge cases** thoroughly
- **Behavioral:** Teamwork and impact examples
- **System design:** Not heavy, basic understanding okay

### Success Metrics

```
Phone Screen:
- Solve problem optimally in 30-35 min
- Discuss approach clearly
- Handle edge cases

Onsite:
- Solve 2 medium-hard problems comfortably
- Explain trade-offs
- Ask clarifying questions
```

## 🍎 Apple (SDE/Hardware/Firmware)

### Interview Format
```
Phone Screen: 1 medium coding problem
Onsite: 2-3 technical rounds (1-2 hours each)
System Design: (For senior roles, less common)
Behavioral: Separate round, culture fit
```

### Apple-Specific

**Unique aspects:**
- Focus on **efficiency** (hardware constraints)
- **Hardware-conscious** thinking (memory, CPU, battery)
- **Code quality** and elegance
- Less emphasis on system design
- More emphasis on **creativity**

**Favorite Topics:**
1. **Arrays/Strings** (25%)
   - Substring, anagram, rotation
   - In-place algorithms

2. **Trees/Graphs** (25%)
   - Traversals
   - Searching
   - Path problems

3. **Linked Lists** (15%)
   - Pointer manipulation
   - Reversals, cycles

4. **Stack/Queue** (15%)
   - Design problems
   - Evaluation

5. **Hash maps** (10%)
   - Frequency, grouping
   - Deduplication

6. **Math/Bit** (10%)
   - Number theory
   - Bit manipulation
   - Apple loves this!

### Preparation (4-5 weeks)

```
Week 1: Arrays + Strings (15 problems)
Week 2: Trees + Graphs (15 problems)
Week 3: Linked Lists + Stack/Queue (15 problems)
Week 4: Hash maps + Math/Bit (15 problems)
Week 5: Mock interviews (3-4), review
```

### Apple-Specific Tips

**During Interview:**

1. **Think about efficiency**
   - "This uses too much memory for a phone"
   - "Can we optimize for battery?"
   - Shows hardware awareness

2. **Code elegance**
   - Clean variable names
   - Minimal complexity
   - Apple values beautiful code

3. **Ask about constraints**
   - "What's the memory limit?"
   - "Will this run on a watch or phone?"
   - Shows practical thinking

4. **Bit manipulation** (more common)
   - Know basic bit operations
   - XOR, AND, OR tricks
   - Apple interviews include more bit problems

5. **Be creative**
   - Apple values novel thinking
   - "What if we approached this differently?"
   - Show problem-solving creativity

### Behavioral Prep

```
Stories that resonate with Apple:
✅ Attention to detail (even small things matter)
✅ Simplicity (elegant solutions vs. complex)
✅ User experience (thinking like user)
✅ Cross-functional collaboration
✅ Shipping quality products
```

### Problem Difficulty

```
Easy: 10-15%
Medium: 70-80%
Hard: 10-20%

Focus on medium, be ready for 1-2 hard problems
```

### Pre-Interview

- [ ] Solved 50+ problems
- [ ] Comfortable with bit manipulation (Apple specific)
- [ ] Code is clean and elegant
- [ ] Mock interviews done (3)
- [ ] Behavioral stories ready
- [ ] Explain efficiency thinking naturally
